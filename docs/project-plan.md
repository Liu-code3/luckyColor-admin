# 项目任务拆解清单

本文档用于管理 `luckyColor-admin` 的本地任务拆解。

这版清单参考了 `vue3-element-admin` 的能力方向，重点把任务拆解调整为更适合学习和落地“多租户 SaaS 后台管理系统”的结构。

本次调整的核心目标：

- 不只做“后台页面”，而是按“平台管理端 + 租户管理端 + 权限体系 + 基础设施 + 多租户隔离”来拆
- 任务颗粒度继续保持最小化，保证一个小任务一个提交
- 强化多租户 SaaS 后台常见能力：租户、套餐、角色、数据隔离、平台侧与租户侧边界
- 补齐更贴近 `vue3-element-admin` 的平台化能力：Mock/在线接口切换、接口文档、生成器、移动端与布局能力、权限颗粒度

说明：

- `docs/` 目录仅本地保留，不纳入 git
- 后续开发仍遵循 gitflow 风格提交：`type: 中文说明`

## 一、参考项目带来的调整点

结合 `vue3-element-admin` 当前 README 中明确展示的能力，本地任务拆解新增或强化了这些学习重点：

- 平台侧与租户侧分层，而不是只做一套通用后台
- 多租户相关不只包含“租户管理”，还要包含套餐、租户状态、租户隔离、平台管理员越权边界
- 权限不只做菜单权限，还要拆成动态路由、按钮权限、角色权限、数据权限
- 基础设施不只做 UI 功能，还要包含接口文档、代码生成器、环境切换、Mock 支持
- 后台页面不只做 CRUD，还要把查询、分页、弹窗、状态切换、导入导出、缓存刷新这些动作拆开

## 二、提交规则

统一格式：

```text
<type>: <中文说明>
```

推荐类型：

- `feat`: 新增功能
- `fix`: 修复问题
- `refactor`: 重构实现
- `docs`: 文档更新
- `style`: 样式调整
- `test`: 测试补充
- `chore`: 构建、脚本、依赖、环境调整

执行原则：

- 一个提交只做一件事
- 一个列表区、一个表单区、一个权限接入点，尽量分别提交
- 一个全局能力的“基础接入”和“页面适配”分开提交
- 一个缓存策略、一种跳转逻辑、一个状态切换，尽量单独提交

## 五、推荐学习顺序

如果目标是“更好掌握多租户 SaaS 后台”，建议按这个顺序学：

1. 先完成 `B2` 到 `B5`
   - 先把动态路由、按钮权限、角色权限、数据权限打稳
2. 再完成 `C2` 到 `C8`
   - 先把传统后台主模块做完整
3. 然后完成 `D1` 到 `D4`
   - 这是 SaaS 化最关键的一层：租户上下文、租户管理、套餐、租户隔离
4. 再完成 `E1` 到 `E8`
   - 这一步是平台成熟度，不只是页面，而是后台工程能力
5. 最后做 `F1`
   - 收口体验、规范和上线能力
