# UI/UX 任务提示词

本文档用于帮助你更高效地给我下达 UI/UX 相关任务，让我更准确地理解你的目标、风格偏好、交付方式和技术约束。

## 1. 最有效的下达方式

推荐你在提需求时尽量同时说明这 8 类信息：

1. 页面或模块
2. 目标用户
3. 核心目标
4. 风格方向
5. 禁忌项
6. 技术约束
7. 参考对象
8. 交付方式

## 2. 推荐提示词模板

你可以直接复制下面的模板来给我下达 UI/UX 任务：

```text
请用 UI/UX 视角优化 [页面/模块名称]。

页面类型：
- 例如：SaaS 后台、管理系统、数据看板、官网、表单页

目标用户：
- 例如：平台管理员、租户管理员、运营、财务、普通用户

核心目标：
- 例如：提升信息层级、增强可读性、突出关键操作、提高转化、减少学习成本

风格方向：
- 例如：企业感、现代感、科技感、极简、编辑感、强对比

禁忌项：
- 例如：不要紫色、不要暗黑、不要拟物、不要复杂渐变、不要过度装饰

技术约束：
- 例如：Vue3 + Naive UI、保留现有交互、不能推翻现有设计系统、必须适配移动端

参考对象：
- 例如：给 2 到 3 个参考站点、截图、组件风格

交付方式：
- 例如：先给设计方向，再改代码；或直接改页面代码
```

## 3. 最推荐的几种提问方式

### 3.1 先出设计方向，再动代码

```text
请先从 UI/UX 角度分析租户管理页，输出信息层级、配色、排版、操作优先级和可优化点，暂时不要改代码。
```

适合场景：

- 页面结构还没想清楚
- 想先讨论方向
- 想先看是否值得改

### 3.2 直接落地代码

```text
请基于现有 Vue3 + Naive UI 技术栈，直接优化角色权限页的 UI/UX，保留现有业务逻辑与交互流程，只调整布局、层级、视觉和体验。
```

适合场景：

- 已经明确要改
- 希望直接出结果
- 更关注落地速度

### 3.3 保留现有风格，小幅增强

```text
请在不推翻现有后台风格的前提下，优化用户管理页的 UI/UX，重点增强按钮层级、表格可读性和空状态体验。
```

适合场景：

- 不想改得太激进
- 项目已有既定视觉语言
- 只想提升完成度

### 3.4 做一版更有设计感的方案

```text
请为工作台首页做一版更有设计感的 UI/UX 方案，风格偏现代 SaaS、强信息层级、轻科技感，但不要花哨和过度动画。
```

适合场景：

- 首页、看板、概览页
- 需要更强品牌感
- 希望有明显升级感

## 4. 如果你想让我理解得更准确

建议你再补充这些信息：

- 当前页面最不好用的点是什么
- 最想优先提升哪 1 到 2 个问题
- 用户在这页最常做的操作是什么
- 哪些内容必须保留不能动
- 哪些部分可以大胆重做

例如：

```text
这页当前最大的问题是信息太平、操作入口不突出、筛选区太拥挤。优先提升列表效率和操作辨识度。现有接口和表格字段不要改，样式和布局可以重做。
```

## 5. 如果你给的是后台管理页面

对后台页来说，最有效的下法通常是：

```text
请优化 [页面名] 的 UI/UX。
目标：
1. 提升筛选区可读性
2. 提升表格扫描效率
3. 突出主操作
4. 优化空状态和异常状态

约束：
- Vue3 + Naive UI
- 保留现有字段和交互逻辑
- 兼容桌面端和移动端
- 不要紫色，不要暗黑
```

## 6. 如果你给的是看板/首页

对首页类页面，更适合这样下：

```text
请优化工作台首页的 UI/UX。
目标用户：平台管理员
目标：更快看到核心指标、待办和风险提醒
风格：现代 SaaS、企业感、轻科技感
禁忌：不要花哨，不要大面积渐变，不要无意义装饰
交付：先分析再改代码
```

## 7. 如果你给的是表单页

对表单类页面，更适合强调：

- 主流程
- 字段分组
- 校验提示
- 保存与返回操作

可直接这样下：

```text
请优化租户创建页的 UI/UX，重点改善表单分组、字段优先级、校验提示和保存操作区，让首次使用的管理员也能快速完成填写。
```

## 8. 最短可用提示词

如果你不想写太多，最少可以这样说：

```text
请用 UI/UX 视角优化租户管理页，保留现有逻辑，重点提升信息层级、主操作按钮和表格可读性。
```

或者：

```text
请先分析这页的 UI/UX 问题，再给我一版可直接落地到 Vue3 + Naive UI 的改造方案。
```

## 9. 我最容易发挥好的任务描述

下面这类描述通常最有效：

```text
请优化角色权限页的 UI/UX。
这是一个 SaaS 后台页面，用户是平台管理员。
目标是让“菜单权限”和“按钮权限”更容易理解和操作。
风格要现代、克制、企业感强。
不要暗黑、不要紫色、不要过度装饰。
技术栈是 Vue3 + Naive UI，保留现有接口和核心逻辑。
请先给出设计方向，再直接落代码。
```
