# 上线检查清单

本文档对应任务 `F1-06`，用于在版本提测、预发布、正式上线前，对 `luckyColor-admin` 做统一核对。

## 1. 构建与依赖

- `pnpm install` 无锁文件冲突
- `pnpm build` 通过
- 若本次改动涉及关键主流程，补跑 `pnpm test:smoke`
- `.env`、`.env.prod`、代理地址、接口前缀已核对
- 新增依赖已确认许可证、体积和必要性

## 2. 登录与权限

- 登录、退出、登录失效回跳正常
- 租户切换后菜单、权限、页面数据不会串租户
- 动态菜单可正常恢复，刷新页面不丢失
- 新增按钮权限、菜单权限、角色授权已完成自测
- 超级管理员与普通租户管理员的可见范围符合预期

## 3. 页面体验

- 列表页加载、空状态、异常状态显示正常
- 抽屉、弹窗、分页、表单校验无阻断问题
- 国际化文案不存在明显未翻译占位
- 暗黑模式、多布局、全屏、水印等平台级能力回归通过
- 移动端与桌面端主要入口均能打开和返回

## 4. 联调与数据

- Mock / 真实接口切换方式已确认
- 关键接口响应结构符合 `A02` 约定
- 错误提示不会重复弹出
- 导入、导出、下载类功能已验证文件名和内容
- 关键写操作有确认弹窗，成功后列表能刷新

## 5. 路由与菜单

- 新增路由能被菜单正确访问
- 隐藏路由不会误出现在导航中
- 菜单同步脚本若有改动，已验证新增节点、排序和父子关系
- 404、iframe、外链、工具页入口均可正常跳转

## 6. 审计与排障

- 高风险操作已按 `AUDIT_LOG_RESERVE_POINTS.md` 预留上下文
- 关键接口失败时能拿到清晰报错
- 若后端返回 `traceId`，已确认排障链路可追踪
- 控制台无明显未处理异常

## 7. 发布前人工核对

- 本次提交只包含本任务相关文件
- `git status` 已确认无误提交的构建产物或临时文件
- 更新了对应任务文档或项目计划勾选状态
- 提交信息符合项目 Gitflow 约定
- 回滚方式已明确，至少知道需回退的提交号

## 8. 建议执行顺序

1. 本地构建与主流程自测
2. 联调环境验证权限、菜单、租户
3. 预发布环境验证真实接口和回归场景
4. 正式上线前再次核对环境变量与菜单数据

## 9. 本阶段结论

`F1-06` 先提供统一上线清单，后续如果引入 CI/CD、E2E、质量门禁或自动化发布脚本，可直接把本文档拆成流水线检查项。
