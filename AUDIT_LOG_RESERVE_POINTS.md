# 日志审计预留点

本文档对应任务 `F1-04`，用于统一 `luckyColor-admin` 的前端审计埋点预留方式，保证后续接入操作日志、行为审计、链路追踪时不用再返工页面结构。

## 1. 目标

- 统一记录“谁在什么租户下，对哪个对象做了什么操作”
- 让列表页、表单弹窗、授权弹窗、导入导出等高风险操作都能挂接审计信息
- 让前端请求链路和后端 `traceId`、`requestId`、操作日志表可以自然关联

## 2. 审计事件最小字段

建议前后端至少围绕以下字段保持一致：

| 字段 | 说明 |
| --- | --- |
| `module` | 模块标识，如 `system:user`、`system:role` |
| `action` | 动作标识，如 `create`、`update`、`delete`、`assign-role` |
| `targetType` | 目标类型，如 `user`、`role`、`menu` |
| `targetId` | 目标主键 |
| `targetName` | 目标展示名称，便于排查 |
| `tenantId` | 当前租户 |
| `operatorId` | 当前登录人 |
| `operatorName` | 当前登录人名称 |
| `requestId` | 前端生成或后端返回的请求唯一标识 |
| `traceId` | 后端链路追踪标识 |
| `result` | `success` / `fail` / `cancel` |
| `reason` | 失败原因或取消原因 |
| `extra` | 变更前后摘要、导入条数、按钮权限数量等扩展字段 |

## 3. 页面级预留点

以下页面建议在对应动作发起前后预留审计上下文：

| 页面 | 建议动作 |
| --- | --- |
| 用户管理 | 新增、编辑、删除、启停、重置密码、分配角色、导入、导出 |
| 角色管理 | 新增、编辑、删除、启停、分配菜单、配置按钮权限、配置数据权限 |
| 菜单管理 | 新增、编辑、删除、显隐切换、状态切换 |
| 部门管理 | 新增、编辑、删除、启停、切换树节点 |
| 字典管理 | 新增、编辑、删除、刷新缓存、导入、导出 |
| 参数配置 | 新增、编辑、删除、刷新缓存 |
| 公告管理 | 新增、编辑、删除、发布、撤回、置顶、取消置顶 |
| 租户管理 | 新增、编辑、删除、启停、切换套餐 |
| 租户套餐 | 新增、编辑、删除、启停、分配菜单、绑定租户 |
| 登录与退出 | 登录成功、登录失败、验证码失败、手动退出、登录失效回跳 |
| 系统工具 | 打开接口文档、生成代码、下载代码包 |

## 4. 前端接入建议

### 4.1 在按钮点击前准备上下文

建议在进入 `confirmAction` 之前就确定审计上下文，至少包含：

```ts
{
  module: 'system:user',
  action: 'delete',
  targetType: 'user',
  targetId: row.id,
  targetName: row.username
}
```

### 4.2 在请求配置中保留扩展位

建议后续统一在请求配置上挂接：

```ts
{
  auditMeta: {
    module: 'system:user',
    action: 'delete'
  }
}
```

当前前端尚未统一消费 `auditMeta`，但新页面或高风险操作应优先按这个结构预留。

### 4.3 成功、失败、取消都要可区分

- 点击确认并请求成功：记为 `success`
- 点击确认但接口失败：记为 `fail`
- 弹窗中主动取消：记为 `cancel`
- 因租户越权、按钮无权限、参数缺失导致未发请求：也应能留下失败原因

## 5. 与现有项目结构的对齐点

- 租户上下文统一从 `src/utils/auth.ts` 的当前登录态获取
- 按钮权限统一复用 `src/constants/permission.ts`
- 请求链路统一经过 `src/utils/http/interceptors.ts`
- 后端响应若返回 `traceId`，建议在拦截器层保留到错误日志或调试面板

## 6. 优先落地顺序

建议后续按以下顺序补真实审计能力：

1. 登录、退出、登录失效
2. 用户、角色、菜单、部门
3. 字典、配置、公告
4. 租户、租户套餐
5. 工具页与导入导出

## 7. 本阶段结论

`F1-04` 先完成“预留点定义”，不直接把埋点 SDK 硬接进页面。后续无论接自研审计接口、Sentry breadcrumb、OpenTelemetry 还是后端操作日志表，都可以沿用本文档的字段和落点。
